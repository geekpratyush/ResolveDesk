Enterprise Support Architecture

ResolveDesk is a professional, high-fidelity customer support ticketing application developed with a decoupled microservice backend in **.NET 8** and a reactive, signal-based dashboard frontend in **Angular 17**.

Bounded Microservices

Identity and TicketCore feature completely decoupled SQLite schemas, communicating asynchronously over event notifications.

Dual EF Core Providers

Runs either SQL Server in staging/prod or zero-install SQLite locally using automated connection string schema mapping.

Angular Signals State

Uses state signals for light UI refresh cycles. Dynamic SLA indicators recalculate instantly based on selected categories.

Clean Architecture Implementation

To maintain high cohesion and prevent cross-cutting infrastructure concerns from bleeding into the core customer workflows, each .NET microservice maintains clear decoupling layers:

1. Core Domain Layer

Holds model entities, business aggregate structures, and repository contracts. The core layer is completely isolated from framework dependencies (no EF Core namespaces or controller contexts).

  • • Ticket.cs: Holds properties (Title, Description, Category, Priority, Status, CreatedAt, ResolvedAt) and TicketResponse child relations.
  • • ITicketRepository.cs: Defines data operations (AddAsync, UpdateAsync, GetByIdAsync).

2. Infrastructure Layer

Implements data persistence, maps SQL schema tables, and configures outgoing networks.

  • • TicketDbContext.cs: Establishes relationships and defines standard tables in SQLite/SQL Server.
  • • TicketsRepository.cs: Concrete repository carrying SQL calls via Entity Framework Core.

3. Presentation Layer (API Endpoints)

Exposes HTTPS routes, processes JWT authorization filters, parses input formats, and handles exceptions.

  • • TicketsController.cs: Handles ticket listing, creating, status modification, and response dispatching.

Self-Healing Startup Data Seeding

To enable immediate demonstration of analytics without manual setup, the Ticket Core microservice runs an automatic validation block in Program.cs:

  1. EnsureCreated() Evaluation: Checks if the target SQLite file (e.g. tickets.db) exists. If missing, it builds the schema tables from scratch.
  2. Empty Check: Checks if the Tickets table contains any records.
  3. 12-Ticket High-Fidelity Dataset Seed: If empty, it seeds 12 distinct tickets spanning Billing, Database, Network, Security, and Software issues. This data features custom date offsets (representing tickets opened over the past week), varying priorities (Critical, High, Medium, Low), and conversation records.
Seeded Ticket Metrics Showcase:

- SLA Calculations: Specific tickets (e.g. "Payment Gateway timeouts") are pre-marked as Closed with a ResolvedAt timestamp set to 3.5 hours after creation, while others have 48-hour offsets. This populates the UI with realistic SLA resolution metrics.

Asynchronous Event Messaging

Microservices maintain independent database records. When a customer submits a ticket, the TicketCore microservice publishes an integration event. The Notification microservice receives this event and acts accordingly.

Message Broker Transport Settings

To simplify local execution and eliminate external dependencies, ResolveDesk configures MassTransit with an **InMemory** loopback bus. This bus processes message consumers in-process instantly when tickets are created or changed.

Angular 17 Signals & SLA Live Analytics

The frontend uses standalone components and Angular Signals to manage state reactively, avoiding the heavy change-detection cycles of legacy Angular modules.

Operational Stats Calculations

When ticket lists update, the dashboard calls `calculateStats()`. This method uses reactive arrays to compute metrics dynamically:

calculateStats() {
this.totalTicketsCount = this.tickets.length;
this.openTicketsCount = this.tickets.filter(t => t.status === 'Open').length;
this.closedTicketsCount = this.tickets.filter(t => t.status === 'Closed').length;
// Calculate average resolution time (SLA)
const resolved = this.tickets.filter(t => t.status === 'Closed' && t.resolvedAt);
if (resolved.length > 0) {
const totalHours = resolved.reduce((sum, t) => {
const diff = new Date(t.resolvedAt).getTime() - new Date(t.createdAt).getTime();
return sum + (diff / (1000 * 60 * 60));
}, 0);
this.averageResolutionHours = Math.round(totalHours / resolved.length);
}
}

Design Patterns Portfolio

To maintain modularity and high performance, ResolveDesk utilizes industry-standard software engineering patterns across both the .NET backend and Angular frontend.

Clean Architecture Boundary

Decouples domain entity objects (e.g. Ticket.cs) from underlying infrastructures. Databases can be migrated or swapped without rewriting business workflows.

API Gateway Pattern (YARP)

Provides a unified, secure gateway URL for client consumption, resolving CORS challenges and abstracting internal service addresses.

Repository Pattern

Exposes collection contracts (interfaces) for data persistence. Separates query assemblies from service behaviors, enabling straightforward mocking in unit tests.

Publish-Subscribe (MassTransit)

Enables reactive microservice actions. Event producers publish messages onto the bus without needing reference to their destination consumers.

Reactive State Stores (Signals)

Maintains high-performance frontend state loops using native Angular 17 Signals. Automatically recalculates metrics (SLA durations, ticket quantities) only when the direct data dependencies change.

Intercepting Filter (HTTP Interceptor)

Interceptors act as pre-processors for client requests, automatically appending authorization headers (Bearer Token) before network dispatch.

How to Launch the Presentation Demo

The entire system is designed for quick startup on any Windows or Linux host machine without Docker:

Option A: Linux / macOS

Assign executable permissions and launch the orchestrator bash script:

chmod +x run-demo.sh
./run-demo.sh

Option B: Windows PowerShell

Run the PowerShell script bypass cmdlet to start all services:

Powershell.exe -ExecutionPolicy Bypass -File .\run-demo.ps1
